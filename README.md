The documentation explains **why** systems have been designed in a particular way, whereas the source code explains **how** they are implemented.

# Documentation Principles

Documentation should explain:

- Why a system exists.
- The design decisions behind it.
- How it integrates with the rest of the project.
- Future planned extensions.

Documentation should avoid duplicating implementation details that already exist clearly within the source code.

---

# Living Documentation

These documents are intended to evolve alongside the project.

Whenever a significant architectural decision is made, the relevant design document should be updated so that future development remains consistent.
